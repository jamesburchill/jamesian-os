# jamesian-os

`jamesian-os` is a public repository for practical operating material from James Burchill: frameworks, prompt packs, checklists, templates, and field-tested thinking for building, reviewing, and running useful systems.

The repo is intended to hold reusable public resources that are useful without requiring access to private client work, internal infrastructure, or proprietary ToolJar code.

## What This Repo Is For

This repo is for public-facing operating kits that help people make better decisions, review their work more carefully, and reduce avoidable risk.

The material here is meant to be:

- practical rather than theoretical
- direct rather than hype-driven
- usable by non-specialists, while still useful to experienced reviewers
- safe to publish publicly
- independent from private projects and credentials

## Current Modules

### The AI Builder Review Kit

[The AI Builder Review Kit](ai-builder-review-kit/README.md) is a practical review framework and prompt pack for people who have used AI-assisted building, vibe coding, no-code, low-code, or AI coding tools to create an app, website, automation, SaaS prototype, script, or internal tool.

Its core idea is simple:

> AI lowers creation friction. It does not eliminate systems complexity.

The kit helps builders review security, data integrity, failure modes, abuse risk, operational readiness, scaling pressure, maintainability, and production readiness before real users depend on what they built.

## What This Repo Is Not

This repo is not a software product, production service, or package library. It is not a replacement for professional security review, legal review, compliance review, or production engineering.

Some resources may help prepare for those reviews, but they should not be treated as certification, formal assurance, or expert sign-off.

## How To Use This Repo

Open the module that matches the problem you are working on, read its README, and use the included prompts, checklists, or templates directly.

For the current module, start here:

- [The AI Builder Review Kit](ai-builder-review-kit/README.md)

## Public-Readiness Rule

Anything added to this repository should be suitable for public release. Do not add secrets, private client details, credentials, internal URLs, private operational notes, or anything that depends on unpublished systems.
